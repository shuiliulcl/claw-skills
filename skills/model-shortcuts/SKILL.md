# Model Shortcuts

当用户想快速切换 DeepSeek 模型时使用本技能。

本技能的目标：
- 用极短中文指令表达"切到聊天模型"或"切到推理模型"
- 优先把用户意图解释为模型切换请求
- 在切换后继续处理同一条任务内容
- 如果当前环境不支持实际切换，则明确告知，并按用户指定的风格继续组织任务

## 适用触发词

以下表达应优先理解为模型切换意图：

### 切到 Chat
- "切聊天"
- "切chat"
- "用chat"
- "普通模式"
- "快速模式"
- "日常模式"

### 切到 Reasoner
- "切推理"
- "切reason"
- "用reasoner"
- "深度分析"
- "复杂模式"
- "思考模式"

## 默认映射

将上述触发词映射为以下模型：

- Chat → `deepseek/deepseek-chat`
- Reasoner → `deepseek/deepseek-reasoner`

## 执行规则

### 1. 只有切换词，没有正文
如果用户只说：
- "切聊天"
- "切推理"

则将其视为单独的模型切换请求。

优先尝试切换会话模型到对应模型，然后简洁确认当前目标模型。

### 2. 切换词 + 正文在同一句
如果用户说：
- "切推理，分析这个 bug"
- "用chat，帮我润色一下"
- "深度分析：下面这段代码"
- "普通模式：帮我总结这篇文档"

则按下面顺序处理：
1. 先识别目标模型
2. 再把剩余内容视为真正任务
3. 优先尝试切换会话模型
4. 切换成功后，继续执行该任务，不要要求用户重复输入

### 3. 如果无法实际切换模型
如果当前环境、版本、权限或命令能力导致无法真正切换模型：
- 不要假装已经切换成功
- 明确说明"未能确认会话模型已切换"
- 但仍按用户指定风格处理任务：
  - Chat 风格：更快、更直接、少做长链推理
  - Reasoner 风格：更重分析、更重拆解、允许更长推导

## 风格约束

### Chat 风格
适用于：
- 日常问答
- 文案润色
- 普通总结
- 常规代码问题

要求：
- 回答直接
- 优先给结论
- 少写长推理链
- 控制篇幅
- 默认追求速度和可读性

### Reasoner 风格
适用于：
- 复杂 bug 分析
- 多步推导
- 数学、算法、架构比较
- 需要更稳妥结论的任务

要求：
- 明确拆步骤
- 先分析约束再下结论
- 允许更长的中间思考结构
- 对不确定点显式标注

## 关键词解析规则

当用户输入以下格式时，冒号或逗号后面的内容视为正文任务：

- "切推理：{内容}"
- "切聊天：{内容}"
- "深度分析：{内容}"
- "普通模式：{内容}"
- "用chat，{内容}"
- "用reasoner，{内容}"

若正文为空，则只执行切换意图。

## 安全回退

如果无法确认模型切换命令是否成功：
- 不要重复尝试多次
- 不要声称已经切换
- 只给出一次简短提示
- 然后按指定风格继续完成用户任务

## 优先级

当用户同时表达"模型偏好"和"任务内容"时：
- 模型偏好优先解释
- 任务内容紧随其后处理
- 不要求用户拆成两条消息

## 示例理解

- "切聊天" → 切到 `deepseek/deepseek-chat`
- "切推理" → 切到 `deepseek/deepseek-reasoner`
- "深度分析：这段崩溃日志" → 推理模型 + 崩溃分析
- "普通模式：润色这句话" → 聊天模型 + 润色任务
- "用reasoner帮我比较两个方案" → 推理模型 + 方案比较
