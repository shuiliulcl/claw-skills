# Lua Code Review 项目约定

> 本文档记录项目中 Lua 代码的特定约定和已知等效写法。
> Code Review 时必须先参考本文档，避免将约定写法误判为问题。
>
> **维护方式**：半墙反馈具体判断问题时，在对应分类下追加条目。

---

## UE 命名空间

| 写法 | 等效写法 | 说明 |
|------|----------|------|
| `UE4.` | `UE.` | 两者在本项目中完全等效，不是错误 |

---

## 分析边界与截断处理

| 场景 | 约定 |
|------|------|
| 脚本输出出现「...及 N 个其他文件」或类似截断提示 | 表明本次 CL 尚有未读取的文件，不得直接断言某方法/功能「未实现」，应改为「在已读文件中未见，可能在未展示的文件中」 |
